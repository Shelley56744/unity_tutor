Chapter 01

Unity 核心介面

初次打開 Unity，先熟悉五大核心視窗。大部分新手問題都能用「選物件 → 看屬性 → 查錯誤」這條流程定位。

Workspace Layout 3 MIN READ
介面名稱主要功能新手注意
Hierarchy 階層顯示目前場景中的所有物件，例如玩家、燈光、模型、Canvas、空物件。藍色方塊通常代表 Prefab。父子物件縮排會影響移動與管理方式。
Scene 場景編輯器視覺化視窗，用於擺放、旋轉、縮放與檢查模型位置。常用 Q、W、E、R 切換工具。
Inspector 屬性設定所選物件的 Transform、Tag、Layer、Collider、Script 與各種 Component。腳本掛載、欄位指定、Is Trigger 開關都在這裡處理。
Project 專案管理所有原始檔案：模型、貼圖、材質、音效、腳本與 Scene。像檔案總管，但 Unity 專案資料夾不要在系統檔案總管內亂搬。
Console 主控台顯示程式錯誤、警告、Debug.Log 訊息，是偵錯核心。先解第一個紅色錯誤，其他錯誤常會連帶消失。

Hierarchy 看結構

確認物件是否存在、父子關係是否正確、玩家或 Canvas 是否被誤刪。

Inspector 改設定

Transform、Collider、Tag、Layer、腳本欄位與 Component 都在這裡調整。

Console 找錯誤

功能失效時先看 Console。錯誤訊息比畫面現象更接近問題根源。

新手工作節奏

選物件 → 看 Inspector → 查 Console → 回 Scene 調整。這四步是 Unity 初學者最重要的排錯節奏。

練習建議

每天開專案先用 Q、W、E、R、F 操作幾分鐘。熟悉後場景擺放速度會明顯提高。

Chapter 03

基礎操作：父子物件與頂點吸附

VR 場景常有大量模型。父子物件用來管理整組物件，頂點吸附用來精準對齊地板、牆面與場景邊界。

Parenting and Snapping 4 MIN READ

父子物件概念

當你需要同時移動上百個物件，例如一整個公園或展場，可以建立 Empty Object 當父容器。移動父物件時，底下子物件會跟著移動。

適合使用時機

場景打包、模型群組、UI 群組、互動物件集合、回憶事件區域，都適合用父容器整理。

建立父子物件步驟

建立空物件

在 Hierarchy 點右鍵 → Create Empty，並改成容易辨識的名稱，例如 Park_Group。

將物件拖入父容器

選取要一起管理的物件，拖到空物件底下；縮排後代表已成為子物件。

移動父物件測試

選取父物件，用 W 移動。確認底下所有子物件都一起移動。

整理位置與命名

大型群組建議命名清楚，必要時記錄 Transform 數值，避免日後交接混亂。

頂點吸附 V 鍵步驟

選取要移動的物件

在 Scene 或 Hierarchy 選取需要精準對齊的地板、牆面或模型。

按住 V 鍵

按住 V 不要放。滑鼠靠近模型頂點時，Unity 會進入頂點吸附操作。

抓住來源頂點

滑鼠移到物件角落或邊緣頂點，按住左鍵拖曳。

拖到目標邊緣

把來源頂點拖到另一個物件的邊緣或頂點，Unity 會自動吸附對齊。

頂點吸附常見卡關

如果 V 鍵吸不到點，通常是 Scene 視角太遠、模型頂點不明顯、選取物件錯誤或 Mesh 結構不適合吸附。

Chapter 04

腳本偵錯：命名禁忌、距離偵測與掛載

Unity C# 新手最常踩的雷：腳本檔名與 class 名稱不一致，或腳本沒有掛到正確物件。

C Sharp Debugging 5 MIN READ

腳本命名的絕對禁忌

C# 腳本檔名必須與程式碼內 public class 名稱完全一致，包含大小寫。檔名是 DistanceTrigger.cs，class 就必須是 DistanceTrigger。

錯誤現象

Inspector 顯示 The associated script can not be loaded，或腳本欄位消失、無法掛載、無法進入 Play Mode。

處理順序

先看 Console 第一個紅色錯誤，再檢查檔名、class 名稱、大括號、分號、拼字與大小寫。

DistanceTrigger.cs
C Sharp 
using UnityEngine;

public class DistanceTrigger : MonoBehaviour
{
    public Transform player;
    public GameObject textCanvas;
    public float triggerDistance = 3.0f;

    void Start()
    {
        if (textCanvas != null)
        {
            textCanvas.SetActive(false);
        }
    }

    void Update()
    {
        if (player == null || textCanvas == null)
        {
            return;
        }

        float dist = Vector3.Distance(player.position, transform.position);
        bool isNear = dist <= triggerDistance;
        textCanvas.SetActive(isNear);
    }
}

建立腳本

在 Project 視窗建立 DistanceTrigger.cs，確認檔名與 class 名稱完全一致。

掛到觸發物件

把腳本拖到觸發點物件，例如手鍊、照片、門口或提示區域。

指定 Player

在 Inspector 將玩家控制器或 Main Camera 的 Transform 拖到 player 欄位。

指定文字 Canvas

將要顯示的文字 Canvas 拖到 textCanvas 欄位，並調整 triggerDistance。

偵錯小技巧

使用 Debug.Log("dist=" + dist) 把距離印到 Console，可以快速確認玩家位置是否正確、距離數字是否如預期變化。

Chapter 05

物理碰撞：Mesh Collider 與 Is Trigger

玩家會穿牆，通常不是 VR 控制器壞掉，而是牆壁、地板或模型沒有正確設定 Collider。

Collider and Trigger 3 MIN READ

Mesh Collider

適合不規則模型，例如牆壁、地板、建築與大型場景模型。會依 Mesh 形狀建立碰撞範圍。

牆壁地板大型模型

Box Collider

適合規則形狀，例如箱子、門、簡單牆面、看不見的阻擋區。效能通常比 Mesh Collider 好。

箱體簡單阻擋隱形牆
Is Trigger 狀態效果適合用途
未勾選具有實體阻擋效果，玩家或其他碰撞物會被擋住。牆壁、地板、欄杆、桌子、不可穿越區域。
已勾選變成感應區，不會阻擋玩家，但可以偵測進入或離開。任務觸發區、提示區、傳送區、事件感應範圍。

穿牆問題快速檢查

要阻擋玩家，Collider 必須存在，而且 Is Trigger 不能勾選。勾選後只會感應，不會阻擋。

選取牆壁或地板模型

在 Hierarchy 或 Scene 選取玩家會穿過的模型。

打開 Inspector

確認物件目前是否已經有 Collider。如果沒有，按下 Add Component。

新增 Mesh Collider

搜尋 Mesh Collider 並加入。模型很複雜時，可改用多個 Box Collider 簡化碰撞。

確認 Is Trigger

要阻擋玩家就保持未勾選；要做感應區才勾選。

Chapter 06

跨場景管理：DontDestroyOnLoad

場景切換時，Unity 預設會清掉目前場景物件。若玩家控制器需要保留，就要使用 DontDestroyOnLoad。

Scene Persistence 3 MIN READ

典型情境

從室內回憶切到戶外公園，如果 OVRPlayerController 被銷毀，玩家視角與控制器會消失。這時可把玩家物件設為跨場景保留。

KeepPlayer.cs
C Sharp 
using UnityEngine;

public class KeepPlayer : MonoBehaviour
{
    private static KeepPlayer instance;

    void Awake()
    {
        if (instance != null && instance != this)
        {
            Destroy(gameObject);
            return;
        }

        instance = this;
        DontDestroyOnLoad(gameObject);
    }
}

建立 KeepPlayer.cs

在 Project 建立腳本，檔名與 class 名稱都使用 KeepPlayer。

掛到玩家根物件

把腳本掛到 OVRPlayerController、XR Origin 或玩家控制器的最上層父物件。

避免重複玩家

新場景如果也放了一個玩家，可能會出現兩個玩家。範例寫法會刪除重複物件。

測試場景切換

從場景 A 切到場景 B，確認玩家視角、控制器、位置與互動功能仍然正常。

不要所有物件都保留

DontDestroyOnLoad 適合玩家、音樂、全域管理器。不適合用來保留每個場景模型，否則記憶體與物件管理會變混亂。

Chapter 07

XR 設定：把專案變成真的 VR

把一般 3D 專案變成 VR 專案的關鍵在 XR Plugin Management 與 XR Origin。本章整理常用步驟與目標平台對照。

XR and Build 4 MIN READ
目標裝置建議 Loader備註
Meta Quest 2 / 3Oculus / Meta XRBuild Target 切 Android，使用 Quest Link 或 APK 安裝。
SteamVR / Valve IndexOpenXR + SteamVR PluginBuild Target 留在 Windows，OpenXR Runtime 設為 SteamVR。
HTC VIVE 系列OpenXR / VIVE Wave桌機接 PC 用 OpenXR；獨立機 VIVE Focus 用 Wave。
Pico 4 / Pico NeoOpenXR + PICO Plugin需安裝 PICO 官方 XR Plugin 套件。

啟用 XR Plugin Management

Edit → Project Settings → XR Plug-in Management，按 Install，再依目標平台勾選對應 Loader。

安裝 XR Interaction Toolkit

Window → Package Manager → Unity Registry，搜尋 XR Interaction Toolkit 並安裝。建議同時安裝 Samples 內的 Starter Assets。

放入 XR Origin

Hierarchy 右鍵 → XR → XR Origin (VR)。場景中原本的 Main Camera 可以刪除，避免兩個攝影機打架。

設定移動方式

常用 Continuous Move 連續移動或 Teleportation 傳送。VR 容易暈時優先用傳送加上 Vignette 視野收縮。

第一次實機測試

Quest 用 Link 線或 Air Link 連線；SteamVR 直接 Play。先測試視角是否正常、控制器是否有顯示、地板高度是否正確。

VR 暈眩注意

新手測試別連續玩超過 15 分鐘。畫面卡頓、加速度過快、頭部跟隨延遲都會放大暈眩。先確保穩定 72 / 90 FPS 再加入特效。

Chapter 08

GitHub SOP：雙重存檔、關閉 Unity、正確 Push

團隊協作的重點是讓下一位同學打開專案不崩潰。Unity 專案上傳前要先存好場景與專案設定，並關閉 Unity。

Version Control 4 MIN READ

雙重存檔

先按 Ctrl + S / Cmd + S 儲存目前 Scene，再點 File > Save Project 儲存專案設定。

關閉 Unity

Push 前務必關閉 Unity。Unity 開著時可能鎖定暫存檔或產生大量 Library 變動。

檢查 GitHub Desktop

確認 Changes 裡出現的是你真的有改的檔案。若出現大量不明暫存檔，先停下檢查。

填寫 Summary

寫清楚本次修改，例如「新增手鍊文字提示」、「修復穿牆」、「加入戶外場景傳送」。

Commit to main

Commit 是把進度打包到本機歷史紀錄。Commit 後還沒有上雲端。

Push origin

Push 才是同步到 GitHub 雲端。Push 完後團隊其他人才能拉到最新進度。

Chapter 09

名詞速查

把英文術語對照中文意思，閱讀官方文件或追蹤 Console 錯誤時不會卡住。

Glossary 2 MIN READ
GameObject場景中所有物件的基底：玩家、燈光、模型、空物件都是。
Component掛在 GameObject 上的功能模組，例如 Transform、Collider、Script。
Prefab預製物件範本。Hierarchy 出現藍色字代表來自 Prefab。
Transform位置、旋轉、縮放三項基本屬性，每個 GameObject 都有。
Tag / Layer用來分類物件，Tag 給程式比對用，Layer 給碰撞與相機篩選用。
Rigidbody物理元件，啟用後物件會受重力與力量影響。
Collider碰撞範圍。沒有 Collider 的物件不會被偵測或阻擋。
Trigger感應型 Collider，不阻擋但會回呼 OnTriggerEnter。
Coroutine可中斷的函式，常用 yield return new WaitForSeconds(1f)。
Scene一個場景檔，存放當下這個畫面的所有物件配置。
XR OriginVR 玩家根物件，包含相機與控制器，是 XR Toolkit 標配。
OpenXR跨平台 XR 標準，Quest、SteamVR、PICO 大多支援。

Chapter 10

FAQ 常見問題

12 題 Unity VR 新手最容易遇到的狀況。遇到錯誤時，先看 Console，再對照這裡的處理方式。

Common Issues 5 MIN READ
1. Inspector 顯示 The associated script can not be loaded 怎麼辦？

先檢查 C# 腳本檔名是否與 public class 名稱完全一致（含大小寫），再看 Console 第一個紅色錯誤。常見原因：少分號、大括號錯誤、class 名稱打錯，或檔名改了但 class 沒改。

2. 玩家為什麼會穿牆或掉出地板？

通常是牆壁或地板沒有 Collider，或 Collider 的 Is Trigger 被勾選。要阻擋玩家請新增 Box Collider 或 Mesh Collider，並確認 Is Trigger 未勾選。

3. Is Trigger 勾選與不勾選差在哪裡？

不勾選時 Collider 會阻擋；勾選時變感應區，可偵測進入與離開但不阻擋。牆壁不要勾，任務觸發區才勾。

4. 為什麼我靠近物件，文字沒有顯示？

檢查距離偵測腳本是否掛在正確物件、player 欄位是否有指定玩家 Transform、textCanvas 是否有拖入、triggerDistance 是否太小，以及 Canvas 在場景中是否可見。

5. V 鍵頂點吸附沒有反應怎麼辦？

先確認有選取要移動的物件，按住 V 不要放，滑鼠靠近模型頂點再拖曳。如果模型頂點不明顯，放大 Scene 視角，或改用 Transform 數值手動對齊。

6. 切換場景後玩家控制器不見了？

玩家控制器需要跨場景保留時，請在玩家根物件掛上使用 DontDestroyOnLoad(gameObject) 的腳本。也要注意新場景不要再放第二個玩家。

7. GitHub Push 前為什麼要關閉 Unity？

Unity 開著可能鎖住暫存檔，或持續更新 Library、Temp、Logs，導致 GitHub Desktop 出現大量不該上傳的檔案。關閉 Unity 後再 Commit 與 Push 比較安全。

8. Commit 跟 Push 有什麼不同？

Commit 是把修改存到本機版本歷史；Push 才是上傳到 GitHub 雲端。只 Commit 不 Push，其他隊友看不到你的修改。

9. 為什麼要 Save Scene 又要 Save Project？

Ctrl + S 主要儲存目前場景；File > Save Project 儲存專案設定與資產狀態。團隊交接前建議兩個都做，降低漏存風險。

10. Hierarchy 物件太多很亂怎麼整理？

建立 Empty Object 當父容器，把同類型物件放在底下，例如 Park_Group、UI_Group、Memory_Triggers。命名清楚會讓後續偵錯與交接容易很多。

11. 戴上 VR 後畫面是黑的或卡在 Splash？

常見原因是 XR Plug-in Management 沒裝、Loader 沒勾、場景沒有 XR Origin 或仍存在多餘的 Main Camera。Build Target 與 OpenXR Runtime 也要對應目標裝置。

12. VR 玩一下子就頭暈怎麼辦？

先確保 FPS 穩定（Quest 至少 72，PCVR 建議 90 以上）。移動方式優先用 Teleportation；若用 Continuous Move 請加 Vignette 視野收縮。每次測試控制在 10–15 分鐘並適度休息。

學習心態

每一個紅色 Bug 都是變熟練的必經過程。遇到錯誤不要慌，先讀 Console，再拆成一個一個小問題處理。
